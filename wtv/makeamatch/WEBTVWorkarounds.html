Image Replacement Problems

//  ********function SeriousLoad explanation...
//	Big work around an image reload problem.  Things work fine the first time.
//	When the game is reloaded, the image replacement problems in WebTV rear their big ugly heads
//	and cause problems especially when using back or rentering the game.
//  The trick is to *fully* load the game twice.
//	The big drawback is that the download time doubles...WAHHHHH!
//  Note that the Restart Button works with this as well. 	 -->


	function SeriousLoad() {
		if (getCookie("reloaded")!="true" ) {
			var today = new Date()
	        var expires = new Date()
			expires.setTime(today.getTime() + 60*60*24*365)
			setCookie("reloaded","true",expires)
			window.status="Loading MatchMate..."
			location.reload()
		}
		else {
		    var today = new Date()
	        var expires = new Date()
			expires.setTime(today.getTime() + 60*60*24*365)
			setCookie("reloaded","false",expires)
		}
	}
	
	At end of file I put this

	<SCRIPT Language="JavaScript">
		SeriousLoad();
	</SCRIPT>

	Another workaround that does not require reloading is called using
	images at render time that you never use for replacement later.  If an
	image is used to layout HTML (i.e. in background or img src tags), don't
	even think of using it as the right hand of an object.src assignment.

Position the yellow selection box as you like it

Leaving white space such as spaces or newlines in anchor tags around images will cause the selection to be a bit larger than your images, especially on the right side. Also, use client side image maps to postion the selection exactly where you want it on images. And you can put anchors in Table TD and TR tags to keep the selection the same width around text and images.

String woofie[i]

WebTV will not index strings like arrays. Use woofie.charAt(i) instead to extract characters.

Using arrays as strings.

WebTV will not convert strings to arrays. Netscape will. pickings = new Array(1,3,4,5,6,7,8,9) woofie = n + ":" + pickings alert (woofie) WebTV stops at the second line. Netscape will display an alert box with the string n:1,3,4,5,6,7,8,9 Workaround: build the string by looping through the array element by element.
