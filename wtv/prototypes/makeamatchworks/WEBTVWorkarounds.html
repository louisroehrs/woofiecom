Image Replacement Problems

//  ********function SeriousLoad explanation...
//	Big work around an image reload problem.  Things work fine the first time.
//	When the game is reloaded, the image replacement problems in WebTV rear their big ugly heads
//	and cause problems especially when using back or rentering the game.
//  The trick is to *fully* load the game twice.
//	The big drawback is that the download time doubles...WAHHHHH!
//  Note that the Restart Button works with this as well. 	 -->


	function SeriousLoad() {
		if (getCookie("reloaded")!="true" ) {
			var today = new Date()
	        var expires = new Date()
			expires.setTime(today.getTime() + 60*60*24*365)
			setCookie("reloaded","true",expires)
			window.status="Loading MakeAMatch..."
			location.reload()
		}
		else {
		    var today = new Date()
	        var expires = new Date()
			expires.setTime(today.getTime() + 60*60*24*365)
			setCookie("reloaded","false",expires)
		}
	}
	
	At end of file I put this

	<SCRIPT Language="JavaScript">
		SeriousLoad();
	</SCRIPT>

	Another workaround that does not require reloading is called using
	images at render time that you never use for replacement later.  If an
	image is used to layout HTML (i.e. in background or img src tags), don't
	even think of using it as the right hand of an object.src assignment.
